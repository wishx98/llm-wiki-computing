*Abstract of the contribution: This contribution provides detailed descriptions and procedures on how to enable the computing service requested by UE via computing NAS for computing site connecting to 6G UPF.*

# 1. Introduction

Based on the solution category table in the agreed pen-holder paper (S2-2601642) and the texts of the pen-holder summary paper (S2-2601619), more detailed principle descriptions and procedures are given on how to enable the computing service requested by UE via NAS signaling for UE connecting with computing site via 6G UPF, which corresponds to 22.2 Solution Variant C: UE Requested Computing Support in 6G by using Computing NAS for Computing Site connecting to 6G UPF.

Also, along with the procedure, some other technical aspects are also illustrated in this paper, including:

22.7, Computing service authorization to UE/AF, subscription based authorization without explicit authorization to UE/AF.

22.11, 22.12, Computing resource monitoring and management by 6G CN.

## 1.1 Architecture and principles descriptions

![](data:image/x-emf;base64...)

Figure 1-1 System architecture of 6G computing service

The overall system architecture of the computing service is illustrated in Figure 1-1.

The CMF is introduced to handle and manage the computing service request initiated by UE and AF. The computing site is introduced to represent the available computing resources in both the 6G CN and DN. When it is located in the 6G CN, it can be co-located with 6G-UPF, or as an entity or a NF behind the 6G-UPF. The awareness of the computing site (its computing resources, location, status) at the CMF can be realized via different ways, e.g., Computing site register to CMF, a proxy NF gather the computing site information and retrieved by CMF, or provided by 3rd party AF (for computing site located in DN).

For the computing service request initiated by UE via NAS message, the following principles apply (22.2):

UE sends the computing service request to the 6G CN (CMF) via NAS message.

The computing resources in the 6G CN and DN are represented as computing site that can provide computing service to the consumer in the form of, e.g., app specific service, pure computing resources and specific AI model.

The awareness of the computing resources (in 6G CN and DN) at CMF can be done via different methods, including:

6G CN: 1) Computing Site registers its resource information to the CMF (either directly, or optionally via a proxy NF), 2) Computing Site registers/reports to the NRF or SMF (if Computing Site is UPF); The proxy NF can gather all the computing site resource information and can be retrieved by CMF.

DN: 1) AF provides the Computing Site information to the CMF (via 6G-NEF), 2) Computing Site sends its information to the CMF via the 6G PSA-UPF that connects to the DN, as described in contribution S2-2601969 (SA2#174).

For both options, the Computing Site information includes, e.g., identification, address, supported computing resource type and capacity/amount, computing site load status and location.

UE connects with the Computing Site via 6G-UPF. The Computing Site can either be co-located with 6G-UPF, or as a standalone entity/NF behind 6G-UPF.

Upon receiving the computing service request from UE, CMF interprets the service KPI, authorizes (may interact with 6G PCF), and selects appropriate Computing Site. The selected Computing Site information is sent back to UE via NAS message.

Based on the received Computing Site information, UE decides to initiate the PDU session establishment or modification procedure towards the 6G-SMF, and sends the uplink data packets to the selected Computing Site via the established PDU session. Alternatively, CMF/6G SMF may also trigger the PDU session establishment or modification procedure for the UE requested service.

The coordination between communication and computing can be done via two different levels:

**Control plane**:

Upon receiving the computing service request, CMF interacts with 6G-PCF for authorization, including the authorization of communication QoS and computing service parameters. Based on this, CMF can select the Computing Site and sends it to 6G-SMF.

The computing delay of the computing service can be estimated by CMF, or reported to CMF by the selected computing site. Then, the communication delay of the same computing service, can be determined by either 6G SMF, PCF or 6G CMF.

6G-SMF selects the 6G UPF based on the authorized communication QoS parameters received from 6G-PCF/CMF, the user plane path quality, and the selected Computing Site for this service.

6G-SMF and CMF can obtain the QoS monitoring results for the computing session, and adjusts the Computing Site, e.g., ask computing site to allocate more computing resources to compensate for the higher transmission delay.

The determined computing service status, e.g., computing delay, transmission rate, result generation rate can also be provided to 6G PCF, 6G SMF for the update of the QoS profile configured to 6G-RAN.

**User plane**:

Computing site sends the computing service related information to the 6G-UPF, including the processing delay, transmission data rate, the computing result generation rate, and the service identifier.

6G-UPF sends the above information to 6G-RAN either via GTP-U signaling, or to 6G-SMF/6G-PCF first with updated QoS profile to RAN, on how to adjust the QoS handling for computing service at 6G-RAN, e.g., prioritize the transmission of the computing service.

Computing site receives the QoS monitoring results from 6G-UPF that related to the computing service, and adjusts the processing progress of the computing task.

For the Computing service authorization to UE/AF, the following principles are summarized (22.7):

CMF authorizes the computing service request received from UE and AF, based on local configuration, subscription information.

For AF initiated computing service request, if it is untrusted AF, the 6G-NEF or a gateway NF will first perform the AF level authorization to authorize whether the AF identity is allowed.

The subscription information for authorization, can either be stored at CMF, or retrieved by CMF from the 6G UDM/UDR. It can include, e.g., allowed UE/AF information, allowed for requesting computing service indication, allowed APP information (e.g., APP Identifier/FQDN/DNN/S-NSSAI), allowed performing area/time.

For the monitoring aspects, the following principles are summarized (22.11, 12):

The QoS monitoring (for communication) result of the computing service, e.g., delay, congestion, can be utilized by the CMF (from 6G SMF, 6G PCF and 6G UPF to CMF), for reselecting the computing site, or adjust the computing resources of the computing site, in order to meet, e.g. the E2E delay.

The CMF sends the computing service and computing site status monitoring request to the computing site, including, e.g., the service status (finished/failed), computing site load status, the processing delay, the transmission rate, computing result generation rate, which are associated with service identifier.

The computing site receives the QoS monitoring result (for communication), from the 6G-UPF or from the CMF, and adjust its computing resources allocated for the computing service.

##### **6.22.2.1.1 Architecture and General Description**

6.22.2.1.1.1 Description for variants A/C/D/E/F/H

A Computing Management Function is introduced to manage the computing resources in 6G CN where the Computing resources are assumed to be located in Computing Site and accessed via 6G UPF. The CS (Computing Site) is assumed to provide various services based on different types of computing resources.

The awareness of its information at CMF can be achieved by computing site registration to the CMF, or optionally via a Proxy NF where the CMF can retrieve. When Computing Site is in DN, the awareness of its information at CMF can be achieved by AF providing to the 6G CN (CMF), or Computing Site itself sending to CMF via 6G PSA UPF.

##### 6.22.2.1.2 Coordination between Communication and Computing

6.22.2.1.2.1 Description for variants A/C/D/E/F/H

The coordination solution for variants A/C/D/E/F/H comprises:

UE/AF requests computing support from 6G CN by using the Computing Request including the computing related information (e.g., Computing Request indication, Task type, overall Task latency, overall Task throughput, Computing resource type, RTT delay requirements, etc.) in order to satisfy the service experience. The computing service request is sent from UE to the 6G CN CMF via NAS message.

Based on the UE/AF request, authorization information in the User Subscription data and Computing Policy (e.g., locally configured or from PCF), the 6G-PCF authorizes both the communication QoS and computing parameters from the service request (sent by CMF) and sends back to CMF. Then CMF selects the appropriate Computing Site while the 6G-SMF further selects the 6G UPF, based on the authorized communication QoS parameters, the selected computing site information, and the UP path quality. The 6G SMF derives the communication QoS profile for RAN and CMF derives the Computing configuration information for Computing Site for Computing Service.

Computing Site for Computing Service enforces the computing configuration information received from the CMF by allocating corresponding computing resources of a Computing Site.

.

The coordination NF variants:

6G SMF derives the communication QoS parameters (e.g., latency, throughput,.)

CMF derives the Computing requirements (e.g., latency, throughput, resource type, etc.)

6G PCF derives both the communication QoS parameters and Computing requirements (e.g., latency, throughput, resource type, etc.)

The coordination decision on how to derive the Computing related parameters in the Computing configuration information in Computing Site can be made based on UE requested information, authorization in User subscription and PCC rule/Computing Policy.

6.22.2.1.3 Computing Site Discovery and (Re-)Selection

For variants A/C/D/E/F/G/H, one of the following NFs is proposed to perform the Computing Site discovery and (re-)selection:

6G SMF (if Computing Site is UPF);

CMF;

NRF.

For variant B, there is no selection of the Computing Site, which is determined by the CS ID associated to the selected UPF. Selection of the computing resources performed by the SHE Controller is out of scope of SA2.

#### 6.22.2.2 Procedure

![](data:image/x-emf;base64...)

Figure X. Procedure for 6G CN based Computing Service Procedure (UE initiated)

.

UE sends the computing service request via NAS message to the CMF, including the UE info, service requirements, e.g., requested resource type (APP specific (in the form of APP Identifier/DNN/S-NSSAI/FQDN), pure computing resources, model info), the RTT delay (or communication delay and computing delay), a profile ID that corresponds to a set of computing resources, result handling instruction and security context. The CMF is selected at this step by the routing/anchor NF.

NOTE: The details of how to route NAS message in step 1 will be investigated under Key Issue 1.

CMF interacts with the 6G-UDM/UDR for authorizing the computing service request. The subscription/authorization information related to computing service stored in the 6G-UDM/UDR includes, e.g., allowed UE/AF information, allowed indication for using computing service, allowed APP information (e.g., APP Identifier/FQDN/DNN/S-NSSAI), allowed area/time (for requesting computing service).

If authorized, CMF further sends the service request parameters (listed in step 1) to the selected 6G-PCF for policy authorization of both communication and computing. If the request is authorized, the 6G-PCF derives the required service parameters for this computing request and sends the authorized computing parameters to CMF, including, e.g., allowed computing resource volume, allowed computing delay, optionally the authorized communication QoS parameters (e.g., delay, error rate, bitrate). The charging rule can also be determined by 6G-PCF for the computing service.

NOTE: The communication QoS parameters for the computing service may only be sent from 6G-PCF to 6G-SMF.

CMF selects the appropriate Computing Site after receiving the authorized service parameters from 6G-PCF, based on its awareness of the available Computing Site information in both the 6G CN and DN, via e.g., computing site registration to CMF, CMF retrieve from a proxy NF that has the computing site information, AF providing computing site information to CMF via 6G NEF, or computing site sending its information to the CMF via 6G PSA UPF. CMF may also determine the offloading options for UE, if only partial offload from UE is allowed.

CMF sends the computing service configuration information to the selected computing site, including the service ID, UE information, requested service parameters of this service (already authorized) and the environment configuration/instalment for running the computing task.

What’s more, CMF can monitor the status of both the computing site and the service, towards this computing site, by sending the service identifier, monitoring parameters (processing delay, transmission rate, computing result generation rate, load status, service status), the notification method (periodic, event) and the target notification address.

After the computing site confirms its readiness at CMF, CMF sends the computing service request response to UE, including the service ID, selected computing site information and allowed computing options for partial offloading.

Based on the received computing site information and the URSP rule, UE initiates the PDU session establishment or modification procedure towards the 6G SMF, including the service ID, PDU session ID, selected computing site information and the corresponding DNN/S-NSSAI. It is assumed that there is a correspondence between the computing site and the DNN/S-NSSAI.

Based on the selected computing site information and the service ID, 6G-SMF retrieves the communication QoS parameters/policy for this service request and selects the 6G UPF accordingly. 6G-SMF configures the 6G-RAN, UE and 6G-UPF for the corresponding QoS parameters and the routing information. 6G-SMF sends the PDU session establishment/modification response message to the UE.

The communication delay may be determined by 6G-CMF/6G PCF based on the subtraction of the processing delay from the overall latency. The processing delay can be determined either by CMF, or by computing site and reported to CMF.

Alternatively, CMF/6G SMF may trigger the PDU session establishment/modification for this UE computing service request.

PDU/Computing session between the UE and the selected computing site is established for transmission of the computing task.

Coordination between communication and computing based on QoS monitoring and computing service/site related monitoring is conducted：

**Control plane**:

6G-SMF and 6G-PCF obtains the QoS monitoring result related to the computing service, e.g., communication delay, congestion, data rate, and sends it to CMF, upon subscription.

CMF determines to adjust the computing site on how to progress the computing service, e.g., allocate more resources to enhance the processing speed, or CMF reselects a computing site to fulfil the E2E delay, given the QoS monitoring result, e.g., excessive communication delay.

CMF monitors the computing site status and computing service information towards the computing site, e.g., load status, processing delay, service status, result generation rate and transmission data rate. These results can be feedback to CMF from computing site, or from 6G UPF. CMF can send it to 6G-SMF and 6G PCF for updating the communication policy (which will be sent to RAN as QoS profile).6G-RAN may adjust the handling of the computing data flow, e.g., prioritize its transmission.

**User plane**:

Computing site sends the computing service monitored results to 6G UPF, including the processing delay, transmission data rate, the computing result generation rate, load status, and the service identifier.

6G-UPF sends the above information to 6G-RAN either directly via N3, or firstly to 6G-SMF/6G PCF and then to 6G-RAN as updated QoS profile. Based on the received computing service monitored results or updated QoS profile, 6G-RAN can treat the QoS flow or computing session accordingly, e.g., prioritize the transmission of the computing service.

Computing site receives the QoS monitoring results from 6G-UPF that related to the computing service, and adjusts the processing progress of the computing task.
