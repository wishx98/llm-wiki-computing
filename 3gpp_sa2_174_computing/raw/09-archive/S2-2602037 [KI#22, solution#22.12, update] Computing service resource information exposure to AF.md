### 6.22.12 Solution #22.12: CP Function (e.g., CSMF, etc.) monitors the QoS for Computing Service from other NFs

#### 6.22.12.0 Topics addressed and High-level Solution Principles

**NOTE (from Pen holder):** This solution category is assumed to only address bullet#22.3b.

This solution addresses bullet#3b of KI#22 and is extracted from Solution # 34 in Annex X.22.

The new CP Function (e.g., CSMF) monitors the configured QoS to ensure that the E2E Computing Service performance requirements (e.g., communication and computing performance requirements) are met.

#### 6.22.12.1 Description

The CSMF reports computing metrics and/or network metrics periodically when the AF subscribes to computing service resource change event. Computing metrics and network metrics include following information.

Computing metric:

GPU memory usage;

GPU compute utilization;

GPU performance

Latency

Throughput

CPU utilization

CPU cores utilization

System memory used

System memory utilization

Communication metric:

UL packet delay, DL packet delay and/or round trip packet delay

#### 6.22.12.2 Procedures

##### 6.22.12.2.1 Support for Computing service resource information exposure to AF procedure

Computing site registers its profile at the CSMF. Computing site profile at least includes computing service ID list it supports, GPU configuration (e.g., model & architecture, Number of GPUs, VRAM capacity), CPU configuration (e.g., CPU architecture, vCPU cores and Clock speed) and memory (e.g., system memory capacity). The computing site reports its computing metric to the CSMF in real time.

Editor’s Note: The definition of computing service is FFS.

The UE registers at 6G network and further establishes a PDU session for control signalling according to 5G-like procedure.

The UE sends computing service registration request to the CSMF via user plane by including UE ID and UE address.

Editor’s Note: Other parameters included in the message is FFS.

The CSMF performs authorization for the request based on operator’s policy and UE subscription data.

If successful, The CSMF sends the response to the UE by including the computing service ID List it supports.

The Application on the UE and AF contacts with application signalling for computing service negotiation.

The AF intends to request a computing task and sends a request message to the CSMF by including AF identifier, UE ID, UE address and computing service ID. The AF also including the subscription to computing service resource change event. If AF is untrusted, the AF may contact with CSMF via NEF. Subscription to computing service resource change event includes computing metric and/or communication metric. Reporting frequency set to "periodic".

The CSMF performs computing task authorization based on operator’s policy and subscription data.

The CSMF sends a computing task response to the AF.

The CSMF determines an E2E QoS requirement according to computing service ID. The CSMF check current network status, e.g., by contacting with NWDAF, to determine transportation QoS can be supported, then The CSMF determines transportation QoS and computing QoS which meet the E2E QoS requirement. The CSMF reserves the Computing resource at selected computing site.

The CSMF sends computing task notification message by including computing service ID and the address of computing site to the UE.

The UE sends acknowledge message to the CSMF.

The CSMF sends transportation QoS requirement to the SMF. If the subscription to computing service resource change includes communication metric in step 7, the CSMF includes the subscription to transportation resource change in the message.

The SMF sends policy authorization request to the PCF by including transportation QoS requirement and the subscription to transportation resource change if applicable. The PCF returns the authorized PCC rule to the SMF.

The SMF may initiate dedicated QoS flow establishment. The QoS monitoring procedure as defined in clause 5.45 of 3GPP 23.501 to retrieve communication latency is enabled in this procedure.

If the subscription to computing service resource change includes computing metric in step 7, the CSMF initiates the computing resource monitoring procedure with selected computing side for the task.

Editor’s Note: The detail of computing resource monitoring procedure is FFS.

The Application on UE, computing site and the AF contact with application signalling for computing execution.

If the event occurs, the SMF sends transportation resource change notification to the CSMF.

if the vent occurs, the computing site sends computing resource change notification to the CSMF.

If the event occurs, the CSMF sends computing service resource change notification to the AF.

AF sends an acknowledge to the CSMF.

#### 6.22.12.3 Services, Entities and Interfaces

CSMF:

Real-time computing resource management;

Authorization of computing service request.

Determination of E2E QoS for computing service, transportation QoS requirement and computing QoS requirement.

Resource reservation for computing service.

Subscription to QoS monitoring for network metrics.

Subscription to computing resource monitoring for computing metrics.

Reporting computing service resource change including computing metric and/or communication metric.

AF:

Request for computing service;

Subscription to computing service resource change event including computing metric and/or communication metric.
