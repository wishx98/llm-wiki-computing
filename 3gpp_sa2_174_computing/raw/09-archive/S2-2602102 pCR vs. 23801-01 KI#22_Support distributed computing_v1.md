# 1. Introduction

This solution is related to Key Issue#22 variant C, and addresses the following bullets:

2. Enablement/authorization of computing service to UE or AF.

a) Whether and how to enable Computing Service.

4. Discovery and (re-)selection of compute site(s) for the Computing Service:

a) Whether and how to discover and (re-)select Computing Site(s) for the Computing Service.

In 3GPP TR 22.870, the 6G network supports selection of multiple SHE for distributed computing has been studied. There are some use case/requirements as following:

*In clause 6.53 (Use case on AI-driven smart factory with computing service), there is a Service Hosting Environment AA near the factory, and a Service Hosting Environment BB that is far from the factory. The smart factory would like to maintain the data security and privacy of its own manufacturing data. So that, the 6G network firstly offloads the task firstly to the Service Hosting Environment AA for data cleansing and partial model training, then transfers the intermediate result to the Service Hosting Environment BB for predictive maintenance analytics. The following requirements*

***[PR 6.53.6-1]*** *Subject to operator policy and regulatory requirement, the 6G network shall be able to support the selection of multiple Service Hosting Environments for 3GPP services and 3rd party services.*

***[PR 6.53.6-2]*** *Subject to operator’s policy and regulatory requirements, the 6G network shall be able to support the coordination amongst multiple Service Hosting Environments for 3GPP services and 3rd party services.*

***[PR 11.22.6-2]*** *Subject to operator’s policy and agreement with the 3rd party, the 6G system shall support to select one or more Service Hosting Environment to fulfil the computing service requirements from 3rd party, considering the UE location, computing capabilities and traffic load, etc.*

This solution proposes to support 6G CN selects computing sites located within 6G CN and DN for distributed computing to meet the computing service requirement.

### 6.22.Y Solution #22.Y: Support distributed computing in CN and DN

#### 6.22.Y.0 Topics addressed and High-level Solution Principles

This solution addresses the following bullets in Key Issue#22:

2. Enablement/authorization of computing service to UE or AF.

a) Whether and how to enable Computing Service.

4. Discovery and (re-)selection of compute site(s) for the Computing Service:

a) Whether and how to discover and (re-)select Computing Site(s) for the Computing Service.

The high-level principles of this solution are as follows:

- The CCF selects a computing site located within 6G CN and a computing site located within DN for distributed computing to meet the service requirement and determines the processing order of computing sites.

- The CCF configures the computing session information (e.g. order of traffic routing) to 6G SMF, and the 6G SMF configures the 6G UPF and computing site with traffic routing rule correspondingly to enable the traffic routing to computing site in CN firstly for prior-processing and then routed to computing site in DN for post-processing.

#### 6.22.Y.1 Description

The new NF, Computing Control Function (CCF) is introduced in 6G CN. The computing site is located within 6G CN (i.e. CS-CN) and/or located within DN (i.e. CS-DN). The new NF, Computing Node is introduced in 6G CN to represent the computing site to report the information of computing resource in computing site, and receives computing enforcement information from CCF.

The UE sends Computing Service Request to CCF via control plane, which includes computing requirements, processing order (computing task is for prior-processing in CN and then post-processing in DN), E2E delay/RT delay (different computing tasks may have different latency requirement) etc. Then CCF selects the CS-CN and CS-DN based on UE’s request, the computing resource related information, network status, UE location etc.

The CCF configures the computing enforcement information to the CS-CN via computing node and CS-DN to indicate the processing order, IP address of information of computing tasks and required computing resource, and provides UE with the information of computing node (CS-CN) and CS-DN and processing order. To enable the traffic routing to CS-CN firstly for prior-processing and then routed to DN for post-processing, the CCF sends computing session information (the information of computing node/CS-CN and CS-DN, and the order of traffic routing) to 6G SMF, and the 6G SMF configures the 6G UPF (i.e. UL CL UPF, PSA UPF) and computing site with traffic routing rule correspondingly.

To support traffic routing among UE, CS-CN, and CS-DN, there are two possible connection models:

The CS-CN connects CS-DN directly.

The CS-CN connects CS-DN via 6G UL CL UPF based on the session breakout model.

In these two models, the UL traffic sent by UE is received at Computing Node (CS-CN) for prior-processing (e.g. partial model training), and then the traffic is sent to CS-DN for post-processing (e.g. data analytics). The CS-DN sends DL traffic (e.g. computing result) to UE.

#### 6.22.Y.2 Procedures

##### 6.22.Y.2.1 Procedure for distributed computing in CN and DN

![](data:image/x-emf;base64...)

Figure 6.22.y.2.1-1: Procedure for distributed computing in CN and DN

NOTE : The intermediate NFs are not shown in the procedure.

The UE sends Computing Service Request to CCF via control plane, and the request includes computing requirements (e.g. computing resource type, computing resource capacity, computing service type etc.), optionally, it includes E2E delay/RT delay (different computing tasks may have different latency requirement), processing order (computing task is for prior-processing in CN and then post-processing in DN) etc..

2a-2b. The CCF authorizes the request for computing service based on the subscription data retrieved from UDM+.

3. The CCF determines more than one computing sites are needed, and selects computing site in CN (i.e. CS-CN) and computing site in DN (i.e. CS-DN) based on the computing resource related information, network status, UE location and UE’s request, e.g. based on processing order in UE request, or due to some computing tasks have low latency requirement, the local processing (select a CS-CN) is required.

4. The CCF sends the computing enforcement information to the CS-CN via computing node and CS-DN respectively, which is to indicate the processing order (e.g. IP address of computing site at next hop), information of computing tasks (e.g. computing task identifier(s), computing task type) and required computing resource. The computing sites (via computing node) respond the computing enforcement response to CCF.

5. The CCF sends computing service response to UE, the response may include the information of computing node (CS-CN) and CS-DN and processing order etc.

6. The CCF sends computing session information to 6G SMF, which includes the information of computing node (CS-CN) and CS-DN, and the order of traffic routing (e.g. indicates the computing node IP address for first routing and then routing to CS-DN IP address for uplink traffic).

7. A session is established among UE, CS-CN (via computing node), and CS-DN based on the connection models illustrated in clause 6.22.Y.1.

Based on received computing session information, the 6G SMF configures the 6G UPF (i.e. UL CL UPF, PSA UPF) and Computing Node with traffic routing rule (e.g. traffic filters) correspondingly.

For the connection model (a), the 6G SMF configures the CS-CN with traffic routing rule for uplink traffic:

- For traffic received at CS-CN from UE, if the Destination IP address and/or port number matches the Computing Node(CS-CN) or CS-DN IP address as indicated in the traffic routing rule, the Computing Node(CS-CN) transfers the traffic to CS-DN.

For the connection model (b), the 6G SMF configures the 6G UL/DL CL UPF with traffic routing rule for uplink traffic:

- For traffic received at 6G UL/DL CL UPF from UE, if the Destination IP address and/or port number matches the Computing Node (CS-CN) or CS-DN IP address as indicated in the traffic routing rule, the 6G UL/DL CL UPF transfers the traffic to Computing Node (CS-CN).

- For traffic received at 6G UL/DL CL UPF from CS-CN, if the Destination IP address and/or port number matches the CS-DN IP address as indicated in the traffic routing rule, the 6G UL/DL CL UPF transfers the traffic to CS-DN.

8. The UE sends uplink packets (e.g. computing tasks) to computing sites. The Destination IP address is either the IP address of Computing Node (CS-CN) or CS-DN, the Source IP address is the UE IP address.

9. The Computing Node (CS-CN) receives the packets from UE. If the Destination IP address is the Computing Node (CS-CN) IP address, the Computing Node replaces the destination IP address by Computing Node (CS-DN) IP address. After computing task is executed, Computing Node sends the packets to CS-DN.

10. The CS-DN executes computing task further and sends computing result to UE.

#### 6.22.Y.3 Services, Entities and Interfaces

**UE:**

- Sends Computing Service Request to CCF, and received Computing Node/Computing Site information from CCF.

**CCF:**

- CCF retrieves the subscription data related to computing service, and authorizes the request for computing service based on the subscription data.

- Selects a Computing node (computing site in CN) and a computing site in DN.

- Sends computing session information (e.g. the information of computing node (CS-CN) and CS-DN, and the order of traffic routing) to 6G SMF.

- Sends the computing enforcement information to the CS-CN via computing node and CS-DN

**Computing Node/Computing Site:**

- Reserves the computing resource based Computing Enforcement Information received from CCF.

- Replaces the destination IP address by CS-DN IP address for UL traffic received from UE, if the Destination IP address is the Computing Node (CS-CN) IP address.

- Executes and computing task and sends the computing result to UE.

**6G SMF:**

- Configures the 6G UPF (i.e. UL CL UPF, PSA UPF) and Computing Node with traffic routing rule (e.g. traffic filters) correspondingly

**UDM+:**

- Stores and provides subscription data related to computing service to CCF.
