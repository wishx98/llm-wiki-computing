# 1. Introduction

This solution is related to Key Issue#22 variant C, and addresses the following bullets:

3. Coordination of communication and computing, service continuity and QoS aspects:

e) Whether and how to support service continuity for computing service upon change of computing site and/or user plane function (e.g. due to UE mobility, computing load balancing, AF influence etc.).

4. Discovery and (re-)selection of compute site(s) for the Computing Service:

a) Whether and how to discover and (re-)select Computing Site(s) for the Computing Service.

Per the following requirement described in 3GPP TR 22.870, the 6G network needs to guarantee minimum interruption during computing site(s) change, and support the selected computing sites switch between those within CN and DN, or within operator controlled DN and third party controlled DN.

***[PR 9.15.6-5]*** *Subject to operator’s policy, the 6G network shall be able to maintain user experience (e.g. for immersive communication) with minimum interruption when the selected computing resources switch between those within the Service Hosting Environment and those in 3rd party computing environment.*

This solution proposes to support network derives and enforces the relocation delay and computing delay to guarantee minimum interruption and meet the service requirement during computing site changes, and support the selected computing sites switch between different networks (e.g. switch from DN to CN).

The relocation delay refers to the time required to complete the instantiation of the computing site and the migration of the context from the source computing site to the target computing site. The VM instantiation typically takes a few seconds to several minutes (e.g. GPU instance instantiation takes over 20 seconds). K8S container instantiation typically takes over 10 seconds, and the cold start for GPU instances takes about 80–120 seconds. Furthermore, the data volume of model training data typically reaches millions of tokens, and it will take several seconds to be relocated from source computing site to target computing site. Therefore, it is important to consider relocation time for guaranteeing the E2E delay.

### 6.22.Y Solution #22.Y: Support service continuity for computing service

#### 6.22.Y.0 Topics addressed and High-level Solution Principles

This solution addresses the following bullets in Key Issue#22:

3. Coordination of communication and computing, service continuity and QoS aspects:

c) Whether and how to improve the service experience.

d) Whether there is a need to improve QoS to satisfy Computing Service requirements.

e) Whether and how to support service continuity for computing service upon change of computing site and/or user plane function (e.g. due to UE mobility, computing load balancing, AF influence etc.).

4. Discovery and (re-)selection of compute site(s) for the Computing Service:

a) Whether and how to discover and (re-)select Computing Site(s) for the Computing Service.

The high-level principles of this solution are as follows:

- The CN can support the selected computing sites switch between same or different networks (e.g. switch from DN to CN), e.g. based on UE requests computing site change from DN to CN.

- The CCF derives the computing site relocation delay and computing delay and provides it to computing site to guarantee meet the service requirement (e.g. E2E delay requirement) during computing site changes,

- The CCF can calculate the minimum required delay and send it to UE, if no suitable target computing site can be selected to meet the delay requirement. The UE may determine either accepts or rejects to continue the computing service.

#### 6.22.Y.1 Description

The new NF, Computing Control Function (CCF) is introduced in 6G CN for computing site (re)selection etc. The computing site is located within 6G CN (i.e. CS-CN) and/or located within DN (i.e. CS-DN). The new NF, Computing Node is introduced in 6G CN to represent the computing site to report the information of computing resource in computing site, and receives computing enforcement information from CCF.

Based on the triggers (e.g. UE mobility, UE requests computing site switch from CS-DN to CS-CN for local processing, load balance), the CCF reselects the computing site/computing node. The source computing site and target computing site can be located in CN or DN (e.g. the selected computing site switch from the DN to CN).

The CCF can derive the relocation delay (and computing delay) to ensure the service requirement is met, e.g. based on the E2E delay and computing delay requirements, and the communication delay (e.g. monitored by CCF), the relocation delay can be derived. The relocation delay refers to the time required to complete the instantiation of the computing site and the migration of the context from the source computing site to the target computing site. Then CCF sends Computing enforcement information to target and source computing site/computing node respectively, which includes the computing site relocation indication, the information of computing site, delay requirement (e.g. computing delay, relocation delay), the information of computing task (e.g. the computing task to be migrated).

In case the CCF can’t select a suitable target computing site can meet the delay requirement, it would calculate the required delay (e.g. required minimum RT delay, required minimum relocation delay) and send the computing site relocation indication and required delay to UE. The UE may determine either accepts or rejects to continue the computing service.

#### 6.22.Y.2 Procedures

##### 6.22.Y.2.1 Procedure for supporting service continuity for computing service

NOTE 1: The intermediate NFs are not shown in the procedure.

1. Computing site reselection is triggered as following:

1a. Due to the UE mobility, UE moves out of service area of serving computing site and CCF receives the notification of UE location.

1b. Due to new requirements received from UE or AF (e.g. UE requests change from CS-DN to CS-CN for local processing, requests lower E2E delay), the serving computing site can’t meet the new requirements.

1c. Due to the change of computing resource status (e.g. load balance or maintenance), the serving computing site is no longer applicable.

2. Based on the triggers in step 1, the CCF determines to reselect the computing site. The source computing site and target computing site can be located in CN or DN (e.g. the selected computing site switch from the DN to CN). The computing node may be also re-selected if reselection of computing site is performed (e.g. the computing node connects to computing site located in CN).

Based on the information of computing task (e.g. computing task type, computing load of computing task, data volume of the computing task to be migrated), computing capability and load of computing site, network status, UE location and service area of computing node/computing site etc, the CCF selects the target computing site and/or computing node, and derives the relocation delay (and computing delay) to ensure the service requirement is met, e.g. based on the E2E delay and computing delay requirements, and the communication delay (e.g. monitored by CCF), the relocation delay can be derived. The relocation delay refers to the time required to complete the instantiation of the computing site and the migration of the context from the source computing site to the target computing site.

*If the CCF can’t select a suitable target computing site can meet the delay requirement (e.g. relocation delay), the step 3 and 4 are performed, and otherwise they are skipped.*

3. The CCF calculates the required delay (e.g. required minimum E2E delay, required minimum relocation delay). The CCF sends Computing Site Relocation Notification to UE, which includes the computing site relocation indication, the required delay.

4. The UE may determine either accepts or rejects to continue the computing service, e.g. if the UE can’t accept the required delay (e.g. required E2E delay, required relocation delay), it terminates the computing service and switches the computing task back to UE.

*If step 3 and 4 are not performed, or the computing service is continued based on step 4, the following steps are performed.*

5a. The CCF sends Computing enforcement information Create to target computing site/computing node, including the computing site relocation indication, the information of source computing site (e.g. IP address), delay requirement (e.g. computing delay, relocation delay), the information of computing task (e.g. the computing task to be migrated). The computing enforcement information is delivered to computing node via the interface between CCF and computing node (e.g. to CS-CN), or via AF (e.g. to CS-DN).

5b. The CCF sends Computing enforcement information Delete Request to source computing site/computing node, including computing site relocation indication, the information of target computing site information, delay requirement (e.g. relocation delay), the information of computing task (e.g. the computing task to be migrated). The computing enforcement information is delivered to computing node via the interface between CCF and computing node (e.g. to CS-CN), or via AF (e.g. to CS-DN).

NOTE 2: How the context transfer between source Computing Site and target Computing Site is done out of SA2 scope.

6. The CCF sends Computing Site Relocation Notification to UE, which includes the computing site relocation indication, target computing site information, source computing site information, information of computing task (e.g. the computing task to be migrated).

7. The UE responds acknowledgement to CCF, and then computing session between UE and target computing node/computing site is established.

#### 6.22.Y.3 Services, Entities and Interfaces

**UE:**

- Receives the Computing Site Relocation Notification from CCF.

- Determines either accepts or rejects to continue the computing service based on the required delay received from CCF.

**CCF:**

- Re-selects the target computing site and/or computing node, and derives the relocation delay (and computing delay) to ensure the service requirement is met.

- Sends the computing enforcement information to source computing site and target computing site.

- Sends Computing Site Relocation Notification to UE, which includes the computing site relocation indication, target computing site information, source computing site information, information of computing task (e.g. the computing task to be migrated).

- Calculates the required delay and provides it to UE, if CCF can’t select a suitable target computing site can meet the delay requirement.
