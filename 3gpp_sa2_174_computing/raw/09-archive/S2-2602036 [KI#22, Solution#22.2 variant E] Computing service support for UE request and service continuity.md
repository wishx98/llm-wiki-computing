# 1. Introduction

This solution address solution#22.2 variant E: UE Requested Computing Support with Computing messages over User Plane

CMF, is defined to manage the computing resource, authorization of the computing service request from UE, determination of E2E QoS requirement, selects computing site and reservice computing resource.

A procedure is introduced to support joint computing and communication scheduling and computing service continuity.

#### 6.22.2.2 Procedure

##### 6.22.2.2.1 Variant E procedures

###### 6.22.2.2.1.1 Support for computing service for UE request procedure

![](data:image/x-emf;base64...)

Figure 6.22.2.2.1.1-1 Support for computing service for UE request procedure

Computing site registers its profile at the CMF. Computing site profile at least includes computing service ID list it supports. The Computing site reports its resource consumption status to the CMF in real time. The CMF aggregates all computing service IDs which are registered by computing sites.

Editor’s Note: The definition of computing service is FFS. The detail of resource consumption status is FFS.

The UE registers at 6G network and further establishes a PDU session for control signalling according to 5G-like procedure. During the procedure, UE and 6G network negotiates the support of computing service and the address of CMF is returned to UE.

Editor’s Note: The details of registration and PDU session establishment are studied in KI#2. The detail of capability negotiation is FFS and it may depend on the study of KI#2.

The UE sends computing service registration request to the CMF via user plane by including UE ID and UE address.

Editor’s Note: Other parameters included in the message is FFS.

The CMF performs authorization for the request based on operator’s policy and UE subscription data.

If successful, The CMF sends the response to the UE by including the computing service ID List it supports.

Application on UE intends to request a computing task and determines the corresponding computing service ID. UE sends computing task request message to the CMF by including computing service ID.

The CMF performs computing task authorization and allocates a task ID for the task. The CMF determines an E2E QoS requirement according to computing service ID. The CMF check current network status, e.g., by contacting with NWDAF, to determine transportation QoS requirement can be supported, then The CMF determines transportation QoS requirement and computing QoS requirement which meet the E2E QoS requirement.

Editor’s Note: How to guarantee transportation QoS requirement between UE and computing site is FFS.

The CMF selects a computing site according to transportation QoS requirement and computing QoS requirement. The CMF contacts with the computing site to reserve computing resource with UE ID, task ID, computing service ID and computing QoS requirement. The computing site reserves the computing resource according to the request.

The CMF sends a computing task response to the UE by including the address of complete site.

The CMF provisions the transportation QoS requirement to the 6G SMF.

The 6G SMF initiates a dedicated QoS flow to the UE if needed. The 6G SMF may contact with 6G PCF for QoS authorization.

Application on UE sends application signalling for computing task execution.

###### 6.22.2.2.1.2 Support service continuity for computing service due to UE mobility

![](data:image/x-emf;base64...)

Figure 6.22.2.2.1.2-1 Support service continuity for computing service due to UE mobility procedure

Communication resources and computing resources are reserved for computing task. Application on UE executes the computing task.

UE performs mobility.

The SMF determines that the serving UPF needs to be changed due to UE mobility.

The SMF sends UPF relocation notification by including the new serving UPF ID to CMF.

The CMF check current network status, e.g., by contacting with NWDAF, to determine transportation QoS requirement can be supported by considering the new serving UPF, then The CMF determines transportation QoS requirement and computing QoS requirement which meet the E2E QoS requirement. CMF may decide to re-select a new computing site according to the updated transportation QoS requirement and computing QoS requirement.

The CMF sends acknowledge message to 6G SMF by including updated transportation QoS requirement and the address of computing site 2 if a new computing site is selected.

The 6G SMF triggers UPF relocation procedure as defined in clause 4.3.5.2, 4.3.5.3 and 4.3.5.4 of TS 23.502.

The UE sends computing task relocation request via the new established PDU session to CMF.

The CMF updates computing resources in computing site1 if computing site is not changed or reserves computing resources in the computing site 2 if computing site is changed.

If computing site is relocated, the computing site 2 may retrieve computing context from the computing site1.

The CMF sends a computing task relocation response to the UE by including the address of the complete site2 if computing site is changed.

The CMF provisions the updated transportation QoS requirement to the 6G SMF.

The 6G SMF initiates a dedicated QoS flow to the UE if needed. The 6G SMF may contact with 6G PCF for QoS authorization.

Application signalling for computing task execution is exchanged.

###### 6.22.2.2.1.3 Support service continuity for computing service due to computing resource change

![](data:image/x-emf;base64...)

Figure 6.22.2.2.1.3-3 Support service continuity for computing service due to computing resource change

Communication resources and computing resources are reserved for computing task. Application on UE executes the computing task.

The CMF detects available computing resource in computing site1 is changed, the CMF re-calculates computing QoS requirement and transportation QoS requirement based on current computing resource status. The CMF may decide to select a new computing site.

The CMF sends computing resource change notification request to 6G SMF by including updated transportation QoS requirement and the address of computing site 2 if compute site is changed.

The SMF determines to update QoS policy according to updated transportation QoS requirement. The SMF may decide to change serving UPF. The 6G SMF may contact with 6G PCF for QoS authorization.

The SMF triggers UPF relocation procedure as defined in clause 4.3.5.2, 4.3.5.3 and 4.3.5.4 of TS 23.502.

The 6G SMF sends computing resource change notification response by including new selected UPF if applicable.

The UE sends computing task relocation request via the new established PDU session to the CMF.

The CMF updates computing resources in computing site 1 if computing site is not changed or reserves computing resources in computing site 2 if computing site is changed.

If computing site is relocated, the new computing site may retrieve computing context from the old computing site.

The CMF sends a computing task relocation response to the UE by including the address of the complete site 2 if applicable.

The 6G SMF initiates a dedicated QoS flow to the UE if needed.

Application signalling for computing task execution is exchanged.
