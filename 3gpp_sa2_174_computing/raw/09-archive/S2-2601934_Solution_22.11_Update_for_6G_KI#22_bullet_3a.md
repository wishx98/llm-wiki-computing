## 6.22 Solutions to KI#22

### 6.22.11 Solution #22.11: CP Function monitors the Computing Resources status in Computing Node/Site

#### 6.22.11.0 Topics addressed and High-level Solution Principles

**NOTE (from Pen holder):** This solution category is assumed to only address bullet#22.3b.

This solution addresses bullet#3b of KI#22 and is extracted from Solution # 5, 19 in Annex X.22.

A CMF (Computing Management Function) is introduced to collect and manage the computing resources of Computing Nodes/Sites in 6G CN.

#### 6.22.11.1 Description

The Reference Point between CMF and Computing Node is named 6G4c.

The CMF can collect the status of computing resources (e.g., computing resource type: GPU or CPU, available computing resource: FLOPS, TOPS, ratio of overall computing resource, other metrics) from the Computing Nodes via Node level procedures (i.e., 6G4c Report) between CMF and the Computing Node. The Node level 6G4c Association needs to be set up between CMF and Computing Node.

NOTE: The detailed computing resources will be defined by the solution for bullet #b of KI#22.1.

![](data:image/x-emf;base64...)

**Figure 6.22.11.1-1: Computing Management Architecture**

One CMF can collect the status of computing resources from multiple Computing Nodes, one Computing Node can report the status of its computing resources to multiple CMFs.

The Computing Node can be:

User Plane Function enhanced with Computing capability;

User Plane Function supported by Computing Sites in 6G CN;

User Plane Function supported by Computing Sites in DN.

The CMF can be a standalone NF or collocated with 6G-SMF.

#### 6.22.11.2 Procedures

##### **6.22.11.2.1 6G4c Association Setup Procedure**

The 6G4c Association Setup procedure is used to setup an 6G4c association between the CMF and the Computing Node, to enable the CMF to use the resources of the Computing Node subsequently to establish 6G4c Sessions. The CMF and Computing Node may exchange the supported functionalities on each side during these procedures.

The setup of a 6G4c association can be initiated by CMF or Computing Node.

The 6G4c Association Setup procedure may be used to request the Computing Node to report the computing resources.

![](data:image/x-emf;base64...)

**Figure 6.22.11.2.1-1: 6G4c Association Setup procedure initiated by CMF**

![](data:image/x-emf;base64...)

**Figure 6.22.11.2.1-2: 6G4c Association Setup procedure initiated by Computing Node**

In the 6G4c Association Setup Request, it can include reporting event trigger, e.g., Date and Time for reporting, reporting period, reporting threshold on available ratio of overall computing resource, overall available computing resource (e.g., FLOPS, TOPS, other metrics), reported resource type (e.g., GPU, CPU, etc.).

##### **6.22.11.2.2 6G4c Association Update Procedure**

The N4 Association Update procedure shall be used to modify an existing 6G4c association between the CMF and the Computing Node. It may be initiated by the Computing Node or by the CMF to update the supported features or available resources of the Computing Node.

The 6G4c Association Update procedure can be initiated by CMF or Computing Node.

![](data:image/x-emf;base64...)

**Figure 6.22.11.2.2-1: CMF initiated 6G4c Association Update procedure**

![](data:image/x-emf;base64...)

**Figure 6.22.11.2.2-2: Computing Node initiated 6G4c Association Update procedure**

In the 6G4c Association Update Request, it can include the updated reporting event trigger, e.g., Data and Time for reporting, reporting period, reporting threshold on available ratio of overall computing resource, overall available computing resource (e.g., FLOPS, TOPS, other metrics), reported resource type (e.g., GPU, CPU, etc.).

##### **6.22.11.2.3 6G4c Association Release Procedure**

The 6G4c Association Release procedure shall be used to terminate the 6G4c association between the CMF and the Computing Node due to e.g., OAM reasons. The 6G4c Association Release Request may be initiated by the CMF or Computing Node.

##### **6.22.11.2.4 6G4c Report Procedure**

The 6G4c Report procedure may be used by the Computing Node to report the status of computing resources supported by the Computing Node.

![](data:image/x-emf;base64...)

**Figure 6.22.11.2.4-1: 6G4c Report Procedure**

The Computing Node detects that an event has to be reported and starts the procedure by sending an 6G4c Report message (Computing Node ID, the type of computing resources, the status of computing resources) to the CMF. The CMF responds with an 6G4c report ACK message (CMF ID). The event parameter contains the name of the event and Computing Node ID. The status parameter contains the actual information the CMF is interested in. If the Computing Node detects that the configured reporting periodicity or reporting threshold is reached on status of computing resources, the Computing Node reports the status of the computing resources to CMF.

The reporting event trigger at Computing Node can be:

Date and Time for reporting;

reporting periodicity;

reporting threshold on available ratio of overall computing resource (e.g., 80%, 90%, etc.);

reporting threshold on overall available computing resource (e.g., FLOPS, TOPS, other metrics);

reported resource type (e.g., GPU, CPU, new processor unit, etc.).

The reported status of computing resources includes one or more of the following:

resource type

available ratio of overall computing resource or overall available computing resource

#### 6.22.11.3 Services, Entities and Interfaces

CMF:

Initiates the 6G4c Association Setup/Update/Release Request.

Responds to Computing Node with the 6G4c Association Setup/Update/Release Response.

Provides reporting event trigger to Computing Node.

Collects and stores the status of computing resources of Computing Nodes.

Computing Node:

Initiates the 6G4c Association Setup/Update/Release Request.

Responds to Computing Node with the 6G4c Association Setup/Update/Release Response.

Stores the reporting event trigger.

Reports the status of computing resources to CMF.
