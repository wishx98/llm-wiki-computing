6.22.5 Solution #22.5: Computing Service Authorization to UE by PCF/CCF based on UE Policy

6.22.5.0 High-level solution Principles

**NOTE (from Pen holder):** This solution category is assumed to only address bullet#22.2a.

This solution addresses the bullet #2a under KI#22.

This solution is extracted from Solution #12, 18, 25 in Annex X.22.

There are two variants:

URSP based (From Solution #12, 18, 25 in Annex X.22) from PCF:

In order to enable the Computing Service to UE, the Computing related UE Policy (i.e., URSP enhanced with new computing related parameter) needs to be provisioned to UE in order to make UE aware whether, where and when it can use the Computing Service to establish the according PDU Session supporting computing. The Computing related UE Policy can be generated based on computing service policy provided by CCF.

Computing Policy based from CCF (from Solution #12 in Annex X.22):

The CCF generates and provides the computing service policy to UE, which can assist UE to decide whether triggers computing service request (e.g. whether is within allowed time and location for requesting computing service) and determines the offloading decision (e.g. which computing task(s) can be offloaded to computing site).

6.22.5.1 Description

Editor’s note: The solution description is FFS.

For variant a:

The CCF generates computing service policy based on computing resource related information. The computing service policy may include allowed time and location for computing service, and guaranteed computing performance etc. The CCF sends the computing service policy to 6G PCF. Then 6G PCF generates Computing related UE Policy based on received computing service policy and send the policy to UE. Based on the Computing related UE Policy, the UE can aware whether, where and when it can use the Computing Service to establish the according PDU Session supporting computing (i.e. request computing service).

For variant b:

The CCF generates computing service policy based on computing resource related information. The computing service policy may include allowed time and location for computing service, and guaranteed computing performance etc. The CCF sends the computing service policy to UE via container, and then UE can decide whether triggers computing service request and offloading decision based on the received computing service policy, for example, whether is within allowed time and location, and guaranteed computing performance meets UE’s requirement.

**6.22.5.2 Procedures**

Editor’s note: Procedures are FFS.

![](data:image/x-emf;base64...)

Figure 6.22.y.2.1-1: Procedure for Computing Service Authorization to UE based on UE Policy

NOTE: The intermediate NFs are not shown in the procedure.

1. The CCF generates computing service policy based on computing resource related information (computing resource type, computing resource capacity, service availability time, service coverage area, computing service type etc.). The computing service policy may include following information:

- Location: Indicates the location where computing service is available or prohibited for use.

- Time window: Indicates the time period when computing service is available or prohibited.

- Guaranteed computing performance: Describes the network (hosted computing site) can guarantee computing capability, computing delay etc.

- Supported computing service types: Describes the network (hosted computing site) can provide computing services, e.g. AI model training, XR rendering etc.

2a. (Optional) The CCF sends the computing service policy to UE directly via container.

2b. (Optional) The CCF sends the computing service policy to 6G PCF. The 6G PCF generates the Computing related UE Policy based on received computing service policy, subscription data, operator policy. The 6G PCF sends the Computing related UE Policy to UE.

3. The UE updates local policy based on received Computing related UE Policy.

4. When UE has computing service requirement (e.g. new application is detected), the UE decides whether triggers computing service request and determines the offloading decision (e.g. which computing task(s) can be offloaded to computing site(s)) by evaluating the Computing related UE Policy, for example, whether is within allowed time and location, whether is guaranteed computing performance meets UE’s requirement.

6.22.5.3 Impacts on Services, Entities and Interfaces

Editor’s note: Impacts are FFS.

**UE:**

- Receives the computing service policy and decides whether triggers computing service request and determines the offloading decision by evaluating the computing service policy.

**CCF:**

- Generates and provides the computing service policy to UE. (Variant a)

- Sends the computing service policy to 6G PCF. (Variant b)

**6G PCF:**

- Generates the Computing related UE Policy based on received computing service policy from CCF. (Variant b)
