## 6.22 Solutions to KI#22

### 6.22.11 Solution #22.11: CP Function monitors the Computing Resources status in Computing Node/Site

#### 6.22.11.0 Topics addressed and High-level Solution Principles

**NOTE (from Pen holder):** This solution category is assumed to only address bullet#22.3b.

This solution addresses bullet#3b of KI#22 and is extracted from Solution # 5, 11, 19 in Annex X.22.

A CMF (Computing Management Function) or CCF (Computing Control Function) is introduced to collect and manage the computing resources of Computing Nodes/Sites in 6G CN.

#### 6.22.11.1 Description

Editor’s note: The solution description is FFS.

For the computing resource host by CS-CN (i.e. computing site located within CN), the computing resource related information can be collected and monitored by CCF via Computing Node. For the computing resource host by CS-DN (i.e. computing site located within DN), the computing resource related information can be provided to CCF via AF.

The computing resource is measured and described as the following parameters:

- Computing Resource Type: Indicates the types of computing resources, such as CPU, GPU, NPU, storage, memory, engine, model, etc.

- Computing Resource Capacity: Specifies the quantity and scale of various computing resources (e.g., number of CPU cores, memory capacity, storage capacity, etc.).

- Computing Service Type: Indicates the types of computing services that can be delivered based on the available computing resources, such as AI model training, XR rendering etc.

- Model Information: Model type, model complexity, and model size etc.

- Address of Compute Node/Computing Site: IP address, FQDN, NF identifier etc.

- Availability Time: Indicates the time periods during which the computing resource is available or prohibited for use.

- Coverage Area: Specifies the locations where the computing resource can be accessed, such as geographic regions, TAI etc.

- Compute Site Status: Describes the current operational condition of the compute site, such as normal operation, maintenance mode, or failure.

- Computing Resource Utilization Status: Describes resource occupancy or load. For example, for CPU/GPU-type resources, this includes the number of used CPU/GPU units or CPU/GPU utilization percentage, for storage-type resources, it includes the percentage of used storage capacity or remaining storage capacity.

#### 6.22.11.2 Procedures

Editor’s note: Procedures are FFS.

##### 6.22.11.2.1 Procedure for collecting the information of computing resource within CN

The procedure is to support to register the computing resource related information within CN to CCF via Computing Node.

**![](data:image/x-emf;base64...)**

Figure 6.22.11.2.1-1: Registration of computing resource related information within CN

1. The Computing Node provides computing resource related information via sending Computing Node Registration Request to CCF. The request includes computing resource related information described in clause 6.22.Y.1.2.

2. The CCF receives the registration request. If the authentication succeeds, the CCF accepts the registration.

3. The CCF sends the Computing Node Registration Response to the Computing Node. The response includes the status of registration (success/failure).

The following procedure is to support to monitor the status of computing resource within CN via Computing Node.

![](data:image/x-emf;base64...)

Figure 6.22.11.2.1-2: Monitor the status of computing resource within CN

1. The CCF sends the Computing resource status Request to Computing Node. The request may include Computing Node identifier, computing resource type, Computing Resource Utilization Status, Compute Site Status, and the detailed description for parameter refers to clause 6.22.Y.1.2. The CCF may also provide the reporting periodicity, event reporting (e.g. the computing resource utilization percentage exceed specific threshold).

2. The Computing Node sends the Computing resource status Report to the CCF. The response includes requested information.

6.22.11.2.2 Procedure for collecting information of computing resource within DN

The procedure is to support AF provides the computing resource related information within DN to CCF.

![](data:image/x-emf;base64...)

**Figure 6.22.11.2.2-1: Collect information of computing resource within DN**

1-2. The SMF subscribes to computing site information notification from the 6G NEF. The SMF may indicate it subscribes which (list of) DNN/S-NSSAI, location etc. The 6G NEF responds the Computing Site information Subscribe Response.

3. The AF sends Computing Site information Create Request. Based on the local policy or SLA, AF provides one or more information of computing resource related information described in clause 6.22.Y.1.2. In addition, the information may also include AF ID, DNN, S-NSSAI, DNAI etc.

4. The NEF authorizes the request.

5-6. The 6G NEF sends the Computing data Store Request to the 6G UDR. The 6G UDR stores the corresponding information and responds to the 6G NEF.

7. The 6G NEF sends Computing Site information Create Response to the AF.

8-9. If there is computing site information available the 6G NEF notifies the CCF. The CCF responds Computing Site information Notify Response.

#### 6.22.11.3 Services, Entities and Interfaces

**CCF:**

- Receives and stores computing resource related information from the Computing Node, authorizes the registration request.

- Requests the subscription of the computing resource status from Computing Node

- Subscribes the computing resource related information of the computing site located within DN from 6G NEF.

**Computing Node/Computing Site:**

- Computing Node (CS-CN) registers the computing resource related information to the CCF.

- CS-DN provides the computing resource related information to the 6G NEF via AF.

**AF:**

- Provides the computing resource related information to the 6G NEF.

**6G NEF:**

- Recieves the computing resource related information to the AF.

- Recieves the subscription from CCF and responds the notification of the computing resource related information of the computing site located within DN.

- Stores the computing resource related information to the 6G UDR.

**6G UDR:**

- Receives and stores computing resource related information from 6G NEF.
