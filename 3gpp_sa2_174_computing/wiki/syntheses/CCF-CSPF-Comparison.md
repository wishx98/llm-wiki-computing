# CCF 与 CSPF 各自的职责和功能

## 核心定义

**CCF（Computing Control Function，计算控制功能）** 与 **CSPF（Computing Service Plane Function，计算服务平面功能）** 是 6G 算力架构中一对紧密协作的网元，分别承担"控制面决策"与"服务面代理执行"的角色。

## CCF：算力控制面的中央决策网元

CCF 的核心职责包括：
- **注册管理**: 代表算力站点受理 CSPF 的注册并存储其能力画像（Profile）。
- **CSPF 选择**: 基于计算服务请求、CSPF 负载或 UE 移动性等内部逻辑执行 CSPF 选择。
- **授权认证**: 对计算服务请求进行授权，查询 UDM 订阅数据。
- **链路管理**: 管理建立 **Computing Layer Link**（计算层链路）以及 6G UPF 与 CSPF 之间的用户面隧道。
- **性能监控与事件设置**: 向 CSPF 设置性能事件（如计算时延、CSPF 负载阈值、任务失败等），并接收事件上报以触发重选或迁移。

## CSPF：核心网与算力站点之间的正向代理/服务面网元

CSPF 代表算力站点与 CCF 和 UE 进行交互，核心职责包括：
- **资源代理注册**: 代表算力站点向 CCF 注册资源信息（如支持的算力类型、xPU 数量、地址等）。
- **任务协调**: 与 UE 在计算层进行协调，动态识别计算任务。
- **QoS 执行**: 向 6G UP 下发下行 QoS 需求。
- **性能监控上报**: 监控正在进行的计算任务性能，并按事件触发或周期性地向 CCF 报告。

### 部署方式
- CSPF 可与 **6G UPF 同址部署**（Option 1），也可位于 UPF 之后（Option 2/3）。
- 算力站点可**内嵌在 CSPF 中**（CN 场景），也可通过自有协议与 CSPF 通信（DN 场景）。

## 协作关系对比

| 维度 | CCF | CSPF |
|------|-----|------|
| 定位 | 控制面中央控制器 | 服务面代理/前端 |
| 核心职能 | 注册管理、授权、选择、策略下发 | 资源代理注册、任务协调、QoS 执行、性能监控 |
| 交互方向 | 向下选择并管理 CSPF | 向上注册并报告状态，向下对接 UE/算力站点 |
| 典型流程 | 接收请求 → 选 CSPF → 建链路 → 设事件阈值 | 注册画像 → 协调 UE → 执行任务 → 上报性能 |

## 架构意义

在华为主导的 S2-2602129 方案中，CSPF 被明确设计为算力站点的"前端（front end）"，使得核心网无需直接面对异构的底层算力基础设施，而是通过标准化的 CSPF 接口实现统一管控，从而解耦通信控制与计算执行。
